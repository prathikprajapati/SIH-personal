Documentation Overview

Welcome to the SecureErase Developer Documentation. This guide provides comprehensive information for integrating our data erasure services into your applications and systems.

Our platform offers RESTful APIs and SDKs for various programming languages to seamlessly integrate secure data erasure capabilities into your workflows.

API Reference

The SecureErase API allows you to programmatically manage data erasure requests, retrieve certificates, and monitor service status.

POST /api/v1/erase
Initiates a data erasure process for specified devices
GET /api/v1/certificates/{id}
Retrieves a specific erasure certificate by ID
GET /api/v1/status
Checks the current service status and availability

Code Examples

Python
import requests
import json

# Initialize API client
base_url = "https://api.secureerase.com/v1"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}

# Create erasure request
request_data = {
    "devices": [
        {"id": "device_123", "type": "ssd"},
        {"id": "device_456", "type": "hdd"}
    ],
    "options": {
        "verification": True,
        "certificate": True
    }
}

response = requests.post(
    f"{base_url}/erase",
    headers=headers,
    json=request_data
)

print(response.json())
JavaScript
const axios = require('axios');

// Initialize API client
const baseURL = 'https://api.secureerase.com/v1';
const config = {
    headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    }
};

// Create erasure request
const requestData = {
    devices: [
        { id: 'device_123', type: 'ssd' },
        { id: 'device_456', type: 'hdd' }
    ],
    options: {
        verification: true,
        certificate: true
    }
};

axios.post(`${baseURL}/erase`, requestData, config)
    .then(response => {
        console.log(response.data);
    })
    .catch(error => {
        console.error('Error:', error.response.data);
    });

Integration Guide

Integrating SecureErase into your application involves three main steps:

  1. Authentication: Obtain an API key from your SecureErase dashboard
  2. Setup: Configure your development environment with the appropriate SDK
  3. Implementation: Call our endpoints to initiate and track erasure processes

Refer to our SDK documentation for language-specific implementations and best practices.

Frequently Asked Questions

Q: What programming languages are supported?

A: We provide SDKs for Python, JavaScript/Node.js, Java, and C#. Our REST API is compatible with any language that can make HTTP requests.

Q: How do I handle rate limiting?

A: Our API implements rate limiting of 1000 requests per hour per API key. Exceeding this limit will result in 429 responses. Implement retry logic with exponential backoff.

Q: Can I customize the erasure process?

A: Yes, our API supports various customization options including verification requirements, certificate generation, and custom logging preferences.
